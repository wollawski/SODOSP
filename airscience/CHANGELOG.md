# CHANGELOG

## [v2.1.0] - 全栈联动集成与 Web 前端深度定制

- **全面整合工作流 (`toweb_2.py`)**：废弃原计划的单独 `bridge.py`，将选型实体转换逻辑与主控引擎紧密融合，实现从网页端几何输入到算风量、选型、算机房、最终呈现 3D 管线的极速一键闭环。
- **修复算法跨目录调用阻断**：在 Web 主程序顶部动态注入 `sys.path.append(os.path.abspath('..'))`，解决前端子目录调用上级排布算法时的 `ModuleNotFoundError`。
- **修复 3D 渲染图表丢失 Bug**：修正了 `steiner_connect_3D.py` 与 `pipeline_2D_to_3D.py` 的数据出口，强制底层绘图后 `return fig`，打通向 Streamlit 网页输出图表的阻滞。
- **全面消除前端乱码与显示截断 Bug**：
  - **单位乱码修复**：完全弃用特殊上标字符，改用 Streamlit 原生 LaTeX 语法 `$m^3/h$` 稳定渲染。
  - **GBK 编码冲突修复**：顶部重新声明 `# -- coding: gbk --` 并全面剔除 Emoji 以适应 Windows 本地编码特性。
  - **长设备名截断修复**：弃用局限性较强的 `st.metric`，注入自定义 HTML/CSS 实现长型号名称自适应换行。
  - **粗体 Markdown 失效修复**：针对 Streamlit 原生提示框无法稳定渲染 Markdown 粗体的问题，启用 `unsafe_allow_html=True` 并使用高仿真 HTML 原生绿框标签绕过渲染异常。

## [v2.0.0] - 工程空间约束与 AI 逻辑漏洞根除

- **强制物理净距约束 (Inflation Method)**：在底层排布引擎的 `_is_rect_valid`（合法性裁判官）与 `_fitness`（适应度打分）中引入虚拟扩大法。通过将房间边界内缩 $1\text{m}$，并将设备角点向外延展 $0.6\text{m}$，在算法底层强行实现了“距墙 $\ge 1\text{m}$、设备间距 $\ge 1.2\text{m}$”的暖通施工刚性规范。
- **算法“摆烂”漏洞彻底修复 (Anti-Loophole)**：修复了 AI 为了规避“大机房面积扣分”而丢弃强制放置（mandatory）设备的漏洞。在适应度函数中增加针对漏放设备的“天价违约金”，并同步下调 `boundary_area_weight` (至 0.005)，使得算法能够完美输出符合规范、空间充裕的机房。
- **算法智能赛马机制 (`--algo auto`)**：升级 2D 组装管道，支持自动模式下后台同步执行 GA 与 PSO。系统依据遗漏设备数优先、利用率次之的评分规则，自动择出优胜方案推送给 3D 环节。
- **修复底层算法命名崩溃**：修正了 GA 排布算法中因形式参数命名冲突导致 `NameError: name 'chromosome' is not defined` 的底层调用错误。

## [v1.2.0] - 多机型寻优

- **设备组合无序寻优 (`select_model.py`)**：摒弃单一型号限制，引入 `itertools.combinations_with_replacement`，支持算法穷举并探索“多型号混合并联”的最优解，极大降低风量冗余。
- **分步逼近与反向复核核心算法**：首创两阶段选型法。第一阶段强制向单台最大风量逼近（锁定初始台数）；第二阶段在匹配设备后，利用所选设备的实际额定风量反除总需求风量，精准卡死选型临界点，大幅节省占地投资。
- **安全余量策略 (`SAFETY_MARGIN`)**：在目标风量侧动态注入 10% 的安全余量缓冲，使智能设备挑选完全契合工程冗余要求。
- **物理数据库映射建立**：在 `EQUIPMENT_DATABASE` 中全面补全从 SXYCK-30 至 SXYCK-300 各大机型的物理长宽高尺寸，打通了业务逻辑向 3D 几何建模生成的数据链。

## [v1.1.0] - 3D 可视化

- **风口数量双轨制计算 (`cal_numofairout.py`)**：针对高大空间，确立了“散湿量/含湿量差需求”与“体积换气次数需求”并行的计算方案，并强制取两者最大值作为总风量标准，杜绝气流死角。
- **3D 面片 Z-Fighting 消除 (`visualize_pool.py`)**：在可视化引擎中，摒弃点阵表示。采用 `Rectangle` 传入真实物理尺寸，并利用 `art3d.pathpatch_2d_to_3d` 定义法线方向 `zdir="z"` 且设定推移高度 `z=0.01`，彻底根除风口图形与地板交错重叠的深度冲突。
